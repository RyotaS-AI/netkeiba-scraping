## 1. 概要

netkeiba（https://race.netkeiba.com）および Yahoo!スポーツナビ競馬（https://sports.yahoo.co.jp/keiba/）の競馬レース関連ページから、出馬情報・オッズ・調教情報・厩舎コメントなどを自動取得し、CSV形式でファイル出力するツールです。

実行時に `race_id`（12桁）を入力することで、対象レースの初回実行か２回目以降かを自動判別し、以下のデータを取得します。
- **初回実行時**: netkeiba（追い切り、厩舎コメント、馬柱データ）および Yahoo競馬（出馬表、各種オッズ）を取得
- **２回目以降**: Yahoo競馬（出馬表、各種オッズ）のみを更新取得

### 動作環境

| 項目 | 内容 |
|------|------|
| OS | Windows 11 |
| 言語 | Python 3.x |
| 実行方法 | コマンドプロンプト（ローカル実行） |

## 2. 機能一覧

実行時に対象レースの `race_id`（12桁）を入力すると、以下のデータ自動取得とCSVファイル生成を行います。
出力フォルダ（`[race_id]_[レース名]`）の存在有無により処理が分岐します。

### フォルダが存在しない場合（初回処理）
**netkeibaから取得（出馬表は出力対象外）**:
| # | 機能名 | 出力CSVのサフィックス |
|---|--------|----------------------|
| 1 | 中間追い切りデータ取得 | `_中間追い切り.csv` |
| 2 | 最終追い切りデータ取得 | `_最終追い切り.csv` |
| 3 | 厩舎コメントデータ取得 | `_厩舎コメント.csv` |
| 4 | 馬柱データ(5走)取得 | `_馬柱データ.csv` |

**Yahoo!スポーツナビ競馬から取得**:
| # | 機能名 | 出力CSVのサフィックス |
|---|--------|----------------------|
| 1 | 出馬表 | `_yahoo_出馬表_yyyymmdd.csv` |
| 2 | 単勝オッズ | `_yahoo_単勝_yyyymmdd.csv` |
| 3 | 複勝オッズ | `_yahoo_複勝_yyyymmdd.csv` |
| 4 | 枠連オッズ | `_yahoo_枠連_yyyymmdd.csv` |
| 5 | 馬単オッズ | `_yahoo_馬単_yyyymmdd.csv` |
| 6 | 馬連オッズ | `_yahoo_馬連_yyyymmdd.csv` |
| 7 | ワイドオッズ | `_yahoo_ワイド_yyyymmdd.csv` |
| 8 | 3連複オッズ | `_yahoo_3連複_yyyymmdd.csv` |
※ `yyyymmdd` は処理を実行した日付が入ります。

### フォルダが存在する場合（２回目以降の処理）
- **netkeiba**: ファイル作成処理なし（スキップ）
- **Yahoo!スポーツナビ競馬**: 初回処理と同様に「出馬表」と「オッズ表」を取得し、既存ファイルを更新（上書き）します。

## 3. 実行方法

### 3.1 事前準備

1. **Pythonのインストール** (Windows 11)  
   [https://www.python.org/downloads/](https://www.python.org/downloads/) からインストール

2. **依存パッケージのインストール**
   ```
   py -m pip install -r requirements.txt
   ```

3. **`.env` ファイルの設定**  
   プロジェクト直下に `.env` ファイルを作成し、netkeibaのログイン情報を記入します：
   ```
   NETKEIBA_LOGIN_ID=登録メールアドレス
   NETKEIBA_PASSWORD=パスワード
   OUTPUT_DIR=output
   ```

### 3.2 実行コマンド

```
py main.py
```

実行後、以下のプロンプトが表示されます：
```
=== 競馬スクレイピングツール ===
対象レースのrace_id(12桁)を入力してください: 
```

対象となるnetkeiba形式の `race_id`（12桁。例: `202606020411`）を入力してEnterキーを押してください。
※URL全体ではなく、12桁の数値のみを入力します。

## 4. 出力仕様

### 4.1 出力フォルダ構成

指定された `race_id` とnetkeibaから取得した「レース名」を使用してフォルダを作成します。
※Yahoo側のレース名によるフォルダ生成は行いません。

```
output/
└── {race_id}_{レース名}/
    ├── {race_id}_{レース名}_yahoo_出馬表_yyyymmdd.csv (Yahoo)
    └── ほか各機能のCSVファイル群（初回処理のみnetkeibaのファイル群）
```

**例 (race_id: 202606020411 の場合)**:
```
output/
└── 202606020411_日本ダービー/
    ├── 202606020411_日本ダービー_yahoo_出馬表_yyyymmdd.csv
    ├── 202606020411_日本ダービー_yahoo_単勝_yyyymmdd.csv
    ├── (中略: その他各種オッズCSV)
    ├── 202606020411_日本ダービー_中間追い切り.csv (初回処理のみ)
    ├── 202606020411_日本ダービー_最終追い切り.csv (初回処理のみ)
    ├── 202606020411_日本ダービー_厩舎コメント.csv (初回処理のみ)
    └── 202606020411_日本ダービー_馬柱データ.csv (初回処理のみ)
```

### 4.2 CSVファイル仕様

- **文字コード**: UTF-8 with BOM（Excelで直接開いて文字化けしない形式）
- **ヘッダー行**: 1行目にカラム名を出力
- **空欄処理**: 値が存在しない場合・`---` ・取消／除外の場合は空欄

### 5.4 馬柱データ(5走)（`_馬柱データ.csv`）

取得ページ: `https://race.netkeiba.com/race/shutuba_past_9.html?race_id={race_id}&rf=shutuba_submenu`

開発用サンプルファイル: `docs/馬柱データ(5走)_開発用サンプル.csv`

各馬のレコードは `[class=HorseList]` ごとに取得します。枠、馬番の値がない場合は全項目をハイフンとして出力します。
過去走（前走〜5走）については、前走を基準に2走〜5走まで同様のルールで取得・出力します。
※サンプルCSVの仕様に準じて「5走の情報」までを取得対象とします（将来的に9走まで拡張することを想定した構成が望ましいです）。

**【基本情報カラム】**  
出力順: `枠, 馬番, 馬名, 今走(前走間隔), 父名, 母父名, 脚質, 斤量`
- **枠**: 枠番。値がない場合はハイフン
- **馬番**: 馬番号。値がない場合はハイフン
- **馬名**: `[class=fc]`内の`[class=Horse03]`から取得
- **今走(前走間隔)**: `[class=fc]`内の`[class=Horse06 fc]`から「中n週」を取得
- **父名**: `[class=fc]`内の`[class=Horse01]`から取得
- **母父名**: `[class=fc]`内の`[class=Horse04]`から取得し、カッコを除外する
- **脚質**: `[class=fc]`内の`[class=Horse06]`配下にある`[class=kyakusitu]`から取得し、言葉を変換して出力（大：逃げ、先：先行、差：差し、追：追込）
- **斤量**: `[class=Jockey]`内の最後のspanに設定されている値を取得

**【過去走情報カラム（前走〜5走の各n走）】**  
（プレフィックスとして「前走」「2走」〜「5走」が付きます。例：`前走日付`、`2走競馬場名`。すべてサンプルCSVの順序に準拠します）
- **n走日付**: その走の列の`[class=Data01]`から日付部分を取得
- **n走(前走間隔)**: 「翌走の日付」から「該当走の日付」の差から「中n週」を求める（例: 前走間隔なら、2走日付〜前走日付の差）。1つ前のレース情報がない場合はハイフン
- **n走競馬場名**: `[class=Data01]`から日付部分のスペースコードの後にある文字を取得（例：中山、京都）
- **n走レース名**: `[class=Data02]`からレース名を取得
- **n走コース(芝/ダート)**: `[class=Data05]`から先頭文字列を取得（例：芝）
- **n走距離**: `[class=Data05]`からコース文字の次にある数値やカッコ書きを含めて取得（例：2200(外)）
- **n走馬場**: `[class=Data05]`からstrongタグにある文字を取得（例：良、重、稍）
- **n走頭数**: `[class=Data03]`の文字列から「〇頭」となる数値を取得し、「頭」をそのまま付与して出力（例：16頭）
- **n走馬番**: `[class=Data03]`の文字列から「〇番」となる数値を取得し、「番」をそのまま付与して出力（例：13番）
- **n走単勝人気**: `[class=Data03]`の文字列から「〇人」となる数値を取得し、数値のみ出力（例：9）
- **n走走破タイム**: `[class=Data05]`から芝や距離の次に記載されているタイム値を取得（例：2:32.4）
- **n走上がり3F**: `[class=Data06]`から4角通過順の次に記載されているカッコで囲まれた数値を取得（例：36.1）。※値が「0.0」の場合はハイフン（`-`）とする
- **n走着順**: `[class=Data01]`から`[class=Num]`の値を取得
- **n走騎手名**: `[class=Data03]`の文字列から、斤量の直前に記載されている文字列（末尾が「頭」「番」「人」でないもの）を取得（例：川田将雅）
- **n走斤量**: `[class=Data03]`の文字列の最後尾にある数字を取得（例：58.0）
- **n走4角通過順**: `[class=Data06]`から先頭の文字列を取得（例：4-4-4-4）
- **n走着差(秒)**: `[class=Data07]`からカッコで括られた数字を取得（例：(0.9)）。値には「秒」の単位を付加する。※対象の馬名（1着馬等）が記載されていない場合、または値がゼロ相当（0.0等）かつ4角通過順がハイフン（`-`）の場合はハイフン（`-`）とする
- **n走場体重**: `[class=Data06]`から最後尾の数字とカッコを含めて取得（例：464(0)）

**【特殊ケース処理（休養など）】**
- **休養セルの取得**: 通常の過去走は `[class=Past]` として出力されますが、休養期間は `[class=Rest]` のセルとして出力されます。データ取得時に `Past` と `Rest` を両方取得し、時系列の順序のずれを防ぎます。
- **休養判定とデータ欠損処理**: `[class=Rest]` のセル、または `[class=Data01]` の値に「休養」の文字があった場合は休養と判定し、過去走の各データ項目（日付や着順などすべて）をハイフン（`-`）とします。
- **休養時の前走間隔**: 休養と判定されたセルの場合、該当走の「n走(前走間隔)」には、セル内の複数の `[class=Data01]` に記載されているテキストをスラッシュ（`/`）で結合した内容（例：`5ヵ月半休養/鉄砲 [0.0.0.3]/2走目 [0.0.0.2]`）を記載します。

## 7. システム仕様

### 7.1 アクセス制御

- **スリープ**: ボット判定回避およびサーバー負荷軽減のため、ページ取得ごとに **3.0〜5.0秒間のランダムなスリープ** を挿入
- **ブラウザ偽装**: 最新PCブラウザ（Chrome等）に偽装した固定の `User-Agent` 指定に加え、`Accept-Language`, `Accept-Encoding`, `Sec-Fetch-*` などの各種リクエストヘッダーを詳細に設定してアクセス
- **ログイン**: `.env` に認証情報が設定されている場合、プレミアムアカウントでログインしてからデータを取得
- **セッション管理**: `requests.Session` を使用してCookie・セッション情報を維持

### 7.2 ログイン仕様

- **ログインURL**: `https://regist.netkeiba.com/account/?pid=login`
- **認証方式**: フォームPOST（login_id, pswd）
- **ログイン後**: 3.0〜5.0秒のランダムスリープ（他のページヘのアクセス時と同様）

### 7.3 race_id のプロバイダ間変換

netkeibaとYahoo競馬では `race_id` の桁数が異なるため、コンソールから入力された12桁のnetkeiba形式のIDから、各プロバイダに適した形式へ自動変換を行ってリクエストを送信します。

**例**:
- **netkeiba向け**: `202606020411`（12桁のまま使用）
- **Yahoo競馬向け**: `2605010811`（先頭の `20` を削除した10桁）

| `main.py` | race_id入力受付・初回/2回目判定と全体ワークフロー管理 |

| `exporter.py` | DataFrame → BOM付きUTF-8 CSV保存 |

| `utils.py` | プロバイダ間の `race_id` フォーマット変換・既存フォルダからのレース名取得（※URL抽出機能は廃止予定） |
| `parsers/shutuba_parser.py` | netkeiba 出馬表HTML の解析（廃止またはレース名抽出のみに特化予定） |
| `parsers/oikiri_parser.py` | netkeiba 中間・最終追い切りHTML の解析 |
| `parsers/comment_parser.py` | netkeiba 厩舎コメントHTML の解析 |
| `parsers/past_race_parser.py` | netkeiba 馬柱データ(5走)HTML の解析（新規作成） |
| `parsers/yahoo_denma_parser.py` | Yahoo競馬 出馬表HTML の解析 |
| `parsers/yahoo_odds_parser.py` | Yahoo競馬 7種類のオッズHTMLの解析 |
| `test_local.py` | ログイン済みHTMLファイル優先の馬柱テスト対応を含む統合テスト |
| `test_past_race.py` | 馬柱データ用の専用単体テストスクリプト（新規作成） |

## 10. テスト実行

### 全体テスト（`test_local.py`）

`docs/` フォルダに格納されたサンプルHTMLを用いたローカルテストです。
今回の機能追加において、**ログイン済みHTMLファイルを優先的に参照する**仕様へ変更されています。

```
py test_local.py
```

出力結果が `output/test_2026_テスト用サンプルレース/` 配下に生成されます。

### 馬柱データ専用テスト（`test_past_race.py` 新規作成）

馬柱機能のみに特化した専用テストスクリプトです。単体でのデバッグや機能確認に利用できます。

```
py test_past_race.py
```
