# 競馬スクレイピング 自動化手順書

## 概要

cron-job.org → GitHub Actions → Google Drive の構成で、PCを閉じた状態でも自動でスクレイピングを実行し、結果をGoogle Driveに保存する。

---

## システム構成

```
cron-job.org が30分ごとにGitHub APIを叩く
        ↓
GitHub Actions の workflow_dispatch が起動
        ↓
run_scheduled.py が schedule.json を読んで
現在時刻に合致するrace_idを実行
        ↓
Google Drive にCSVを保存
```

---

## 初期設定（一度だけ実施）

### 1. Google Cloud の設定

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. **Google Drive API** を有効化（APIとサービス → ライブラリ → Google Drive API）
3. **OAuth同意画面** を設定（APIとサービス → OAuth同意画面）
   - User Type: 外部
   - アプリ名・メールアドレスを入力
   - 対象 → テストユーザーに自分のGmailアドレスを追加
4. **OAuth クライアントID** を作成（APIとサービス → 認証情報 → 認証情報を作成）
   - 種類: デスクトップアプリ
   - JSONをダウンロードして `credentials.json` にリネームし、プロジェクトフォルダに配置

### 2. リフレッシュトークンの取得（ローカルで一度だけ実行）

```bash
pip3 install google-auth-oauthlib
python3 get_refresh_token.py
```

ブラウザが開くので自分のGoogleアカウントでログイン。
ターミナルに表示される3つの値をメモする。

```
GOOGLE_OAUTH_CLIENT_ID:     xxx
GOOGLE_OAUTH_CLIENT_SECRET: xxx
GOOGLE_OAUTH_REFRESH_TOKEN: xxx
```

### 3. Google Drive の設定

1. Google Drive で出力先フォルダを作成
2. フォルダのURLからIDをメモ（`https://drive.google.com/drive/folders/【ここ】`）

### 4. GitHub リポジトリの作成

```bash
cd /[プロジェクトのパス]/netkeiba_scraping
git init
git branch -m main
git add .
git commit -m "initial commit"
git remote add origin https://github.com/[ユーザ名]/netkeiba-scraping.git
git push -u origin main
```

### 5. GitHub Secrets の登録

リポジトリの Settings → Secrets and variables → Actions で以下を登録：

| Secret名 | 値 |
|---|---|
| `NETKEIBA_LOGIN_ID` | netkeibaのログインID |
| `NETKEIBA_PASSWORD` | netkeibaのパスワード |
| `GOOGLE_OAUTH_CLIENT_ID` | 手順2で取得した値 |
| `GOOGLE_OAUTH_CLIENT_SECRET` | 手順2で取得した値 |
| `GOOGLE_OAUTH_REFRESH_TOKEN` | 手順2で取得した値 |
| `GOOGLE_DRIVE_FOLDER_ID` | 手順3でメモしたフォルダID |

### 6. GitHub PAT（Personal Access Token）の作成

1. GitHub右上のアイコン → Settings → Developer settings
2. Personal access tokens → Fine-grained tokens → Generate new token
3. 設定：
   - Repository access: `netkeiba-scraping` のみ
   - Permissions → Actions: Read and Write
4. 生成されたトークンをメモ（一度しか表示されない）

### 7. cron-job.org の設定

1. [cron-job.org](https://cron-job.org) でアカウント作成
2. CREATE CRONJOB で以下を設定：

**URL:**
```
https://api.github.com/repos/[ユーザ名]/netkeiba-scraping/actions/workflows/scrape.yml/dispatches
```

**Execution schedule:** Every 30 minutes

**Advanced → Request method:** POST

**Advanced → Headers:**

| Key | Value |
|---|---|
| `Accept` | `application/vnd.github.v3+json` |
| `Authorization` | `Bearer github_pat_xxx（手順6のトークン）` |
| `Content-Type` | `application/json` |

**Advanced → Request body:**
```json
{"ref":"main"}
```

## 毎週の使い方

1. `schedule.json` に当週のrace_idと実行時刻（JST）を記入

```json
[
  {"race_id": "202506010811", "jst": "08:00"},
  {"race_id": "202506010812", "jst": "08:00"},
  {"race_id": "202506010901", "jst": "14:00"}
]
```

2. GitHubにpush

```bash
git add schedule.json
git commit -m "schedule更新"
git push
```

3. 設定した時刻（±30分以内）に自動実行され、Google DriveにCSVが保存される

---

## 手動実行

GitHub → Actions → 競馬スクレイピング → **Run workflow** → race_id（12桁）を入力 → 実行

race_idを空欄にするとschedule.jsonの内容を即時実行できる。

```
├── requirements.txt         # Pythonパッケージ一覧
├── .github/workflows/

└── docs/
    ├── 仕様書.md
    └── 自動化手順書.md（本ファイル）
```
